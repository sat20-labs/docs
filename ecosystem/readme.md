# Ecosystem：生态建设

聪网生态需要 BTC 社区、开发者、节点运营者、钱包、交易平台、Indexer、Explorer、做市商、AI Agent 构建者、研究者和内容贡献者共同参与。

生态页面的目标不是重复协议说明，而是让外部建设者快速判断：聪网现在能做什么、我可以参与什么、下一步如何进入测试网或合作流程。

## 第一生态主张

**让每一个 BTC 社区拥有自己的金融基础设施。**

一个 BTC 社区可以逐步摆脱对外部交易平台、外部浏览器和外部钱包入口的单一依赖。聪网帮助社区拥有自己的 Core Node、Indexer、Explorer、钱包、Launchpad、AMM、限价单、DAO、社区基金、DEX 前端与后台，并通过 AI Agent 降低部署和运营门槛。

## 为什么在聪网上构建

1. 资产来自比特币主网。
2. Indexer 将多协议资产统一表达为可验证的资产事实。
3. STP 提供可验证的用户控制权。
4. 聪网提供更低成本、更快确认和智能合约环境。
5. GAS 为合约执行和生态激励提供经济入口。
6. AI Agent 可以降低复杂协议的使用门槛。

## 生态增长飞轮

1. Indexer 统一表达 BTC L1 多协议资产。
2. STP 让 BTC 资产安全进入聪网。
3. 智能合约创造使用场景。
4. GAS 为网络资源定价。
5. 钱包和 Agent 降低使用门槛。
6. 交易平台、Indexer 和 Explorer 提供流动性与可见性。
7. 更多资产和应用进入聪网。

## 战略合作

投资机构、研究机构、生态基金、交易平台、钱包、基础设施服务商和区域社区可以作为战略合作伙伴参与聪网。战略合作的重点不是短期资产叙事，而是围绕真实使用、基础设施、流动性、开发者、社区试点和长期公共品建设展开。

## 当前重点

| 重点 | 说明 |
| --- | --- |
| 社区试点 | 帮助首批 BTC 社区搭建 DEX / DAO / Explorer / 钱包入口 |
| EVM 开发体验 | 让 Solidity 开发者能部署、调用、验证第一个合约 |
| 流动性基础 | AMM、限价单、Launchpad 和跨社区交易场景 |
| Agent Wallet & Safety | 让 AI Agent 能验证资产安全并通过钱包授权操作 |
| Community Builder Agent | 让社区通过对话规划基础设施、参数、合约和部署流程 |
| 数据与证据 | 用 Indexer、Explorer、交易和测试网演练证明每一步 |
