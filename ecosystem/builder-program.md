# Builder Program

Builder Program 用于吸引和支持聪网生态建设者。它服务于长期公共品、真实应用和可验证基础设施，而不是短期活动。

## 当前优先招募

| 方向 | 我们希望看到什么 |
| --- | --- |
| BTC 社区试点 | 愿意在测试网部署社区 DEX、DAO、Explorer 和钱包入口的社区 |
| EVM 开发者 | 能把 AMM、限价单、Launchpad、支付、稳定币等合约跑到聪网上的团队 |
| 基础设施团队 | Core Node、L1/L2 Indexer、Explorer、RPC、监控和部署工具 |
| 钱包 / 交易平台 | 集成 BTC L1 与聪网充值提现、资产查询、STP 状态和授权流程 |
| AI Agent 开发者 | Agent Wallet & Safety、Community Builder Agent、合约助手和部署助手 |
| 流动性伙伴 | AMM 池、限价单市场、跨社区流动性和做市工具 |
| 安全与审计 | STP、合约、钱包 adapter、Indexer 和跨层状态验证 |
| 内容与区域社区 | 教程、演示视频、翻译、社区运营和本地化开发者支持 |

## SAT20 Labs 能提供什么

| 支持 | 说明 |
| --- | --- |
| 开源代码 | Indexer、SatoshiNet、STP、钱包 SDK、合约与工具的公开源码入口 |
| 测试网资源 | 测试节点、测试资产、测试 GAS、Explorer 与可复核交易证据 |
| 技术接入 | 架构评审、接口定位、测试流程、风险边界和调试支持 |
| 合约模板 | AMM、限价单、Launchpad、DAO、通道合约和后续模板 |
| 文档与内容 | 接入指南、案例页、教程、演示脚本和发布材料 |
| 集成支持 | 钱包、Explorer、Indexer、DEX 前端和社区后台接入 |
| 伙伴引荐 | 在资源允许时连接流动性、钱包、交易平台、安全和区域社区伙伴 |

部分支持是公开文档和开源代码，任何人都可以直接使用；更深入的联合开发、部署支持、流动性协作和市场发布需要进入合作评估。

## 评估标准

1. 是否增强用户资产控制权。
2. 是否降低开发者接入成本。
3. 是否提升聪网资产流动性。
4. 是否形成可复用公共品。
5. 是否有测试网或主网可验证结果。

## 如何申请

申请材料应尽量包含：

1. 项目目标。
2. 团队背景和负责人与联系方式。
3. 面向的用户或社区。
4. 需要聪网提供的模块：Core Node、Indexer、Explorer、Wallet、STP、合约、DEX、DAO、Agent 或其他工具。
5. 当前状态：想法、原型、测试网、主网、已有用户或已有流动性。
6. 计划在测试网完成的第一步。
7. 已有代码、文档、交易、合约、截图或演示视频。
8. 风险边界和需要 SAT20 Labs 协助确认的问题。

可通过以下入口联系：

- X：[SAT20Labs](https://x.com/SAT20Labs)
- GitHub：[sat20-labs](https://github.com/sat20-labs)
- 官网：[sat20.org](https://sat20.org)

## 合作流程

1. 提交项目资料和当前需求。
2. SAT20 Labs 确认角色、模块、风险边界和测试网目标。
3. 双方确定最小可验证里程碑。
4. 项目进入测试网部署或集成。
5. 用 Explorer、Indexer、交易、合约和演示记录验证结果。
6. 通过后进入 [Built on SatoshiNet](built-on-satoshinet.md) 或后续社区展示。

## 项目展示资料

每个生态项目进入展示页前，建议提供：

1. 项目解决什么问题。
2. 使用了聪网哪些模块。
3. 当前状态。
4. 在线体验或测试网入口。
5. 开源地址。
6. 合约、交易或通道证据。
7. 团队和联系方式。
8. 希望获得哪些合作。
