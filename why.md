# 为什么比特币需要开放的执行层

Bitcoin L1 优先保障简单、可验证、稀缺资产和最终结算。它不适合承载所有高频交易、复杂合约、社区治理和应用执行。随着 BTC 原生资产、Ordinals、Runes、BRC20、ORDX 和更多应用需求出现，执行迁移到其他层是自然趋势。

真正的问题不是“是否需要扩展执行”，而是：

1. 扩展层是否仍能追溯 BTC L1 的资产事实。
2. 用户是否保留异常情况下的退出和保护路径。
3. 社区是否可以独立运行自己的基础设施。
4. 开发者是否可以在 BTC 原生资产之上构建应用。
5. 网络是否避免重新回到中心化托管模型。

聪网是对这一问题的一种开放实现。它不是要替代 Bitcoin，而是让 Bitcoin 原生资产获得一个更适合应用的执行环境。

## 聪网的路径

1. Bitcoin L1 提供资产来源和最终边界。
2. Indexer 提供可查询、可复核的资产事实。
3. STP 提供跨层控制、退出和旧状态惩罚路径。
4. SatoshiNet 提供更快、成本更低、可编程的执行环境。
5. 钱包和 AI Agent 将复杂协议变成可验证、可授权的用户操作。

## ORDX 的位置

ORDX 是 SAT20 体系中的聪本位资产协议。它的历史和协议价值应在 [资产发行协议](protocol/indexer/asset-issuance.md) 中理解，而不是作为全站唯一主叙事。聪网的定位更宽：它面向 BTC L1 多协议资产、跨层控制、开放执行、节点网络和社区自有基础设施。
