# SAT20 Docs V3 重构规划

本文用于规划 `docs.sat20.org` 的 V3 内容结构。它不是协议白皮书，而是 Docs 本身的产品设计文档。

## 新定位

`docs.sat20.org` 是聪网开放网络的协议事实、开发接入、安全验证和实现细节的权威来源。官网负责结果、机会和行动路径；Docs 负责实现、证据和风险边界。

Docs V3 的目标是：

> 看见机会 → 选择角色 → 验证当前能力 → 完成第一次操作 → 运行自己的基础设施 → 申请合作 → 成为长期建设者。

## V3 相比上一版的关键变化

1. 聪网定位从“SAT20 Labs 提供社区工具”升级为“开放的比特币原生执行网络”。
2. 新增运行网络，承接挖矿节点、核心节点、Indexer、Explorer、RPC、监控和去中心化路线。
3. 新增网络经济，区分 Gas Unit、GAS 资产、费用流、节点质押、分配草案和设计中问题。
4. 新增治理与支持，区分 SAT20 Labs、未来基金会、SIP、协议开发支持和安全披露。
5. 收敛公开导航中的低内容占位页，未达到可运行标准的页面保留为规划项，不占据主路径。
6. SatoshiNet Today 改为三维状态：实现状态、可用环境、文档证据。
7. 路线图改为 Now / Next / Later，不再同时维护重复的阶段叙事。

## 页面进入公开主路径的最低标准

页面进入 `SUMMARY.md` 前，尽量满足：

1. 当前状态和适用网络。
2. 可访问的产品、RPC、Faucet、Explorer 或仓库入口。
3. 操作前准备。
4. 实际步骤或命令。
5. 预期结果。
6. Explorer / Indexer 验证方式。
7. 当前限制。
8. 常见错误和恢复方式。
9. 最后验证日期和版本。

不满足最低标准的页面可以保留为规划项，但不放入用户主路径。
