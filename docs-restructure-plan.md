# SAT20 Docs 重构规划

本文用于规划 `docs.sat20.org` 的新内容结构。它不是协议白皮书，而是 docs 本身的产品设计文档。

## 新定位

`docs.sat20.org` 是聪网生态的公共知识库、开发者入口和信任入口。它解释 SAT20 为什么存在，聪网如何继承比特币安全性，indexer 如何统一表达 BTC L1 多协议资产，STP 如何保护用户资产，智能合约和 GAS 如何形成应用生态，以及 AI Agent 如何帮助用户安全操作复杂协议。

技术白皮书只是其中一部分。更重要的是让不同角色知道自己为什么加入、如何加入、如何验证风险。

## 目标读者

| 读者 | 需求 | 文档任务 |
| --- | --- | --- |
| 比特币资产用户 | 资产是否安全，能否退出 | 解释 STP、承诺交易、惩罚交易、强制关闭 |
| 铭文、Runes、BRC20、ORDX 玩家 | 资产能否进入聪网，在哪里交易 | 提供跨层、交易、验证和风险指南 |
| 开发者 | 如何构建应用 | 提供快速开始、合约、Indexer、SDK 和样例 |
| 钱包和交易平台 | 如何接入充值提现和资产证明 | 提供集成指南、API、风控和状态模型 |
| Indexer / Explorer 团队 | 如何表达 BTC L1 与聪网 L2 的资产事实 | 提供 L1/L2 状态模型、API、异常处理、L2 节点内置索引和 L1 事实分布式验证路线 |
| AI Agent 开发者 | 如何让 Agent 操作资产但不接触私钥 | 提供 skill、adapter contract、安全验证矩阵 |
| 生态伙伴和投资机构 | 网络为什么有长期价值 | 提供 thesis、路线图、GAS 经济和生态机会 |

## 信息架构

```text
欢迎来到 SAT20
Learn：理解聪网
Use：使用聪网
Build：开发者中心
Protocol：协议与白皮书
AI Agent：可验证的钱包自动化
Ecosystem：生态建设
Roadmap：路线图
Resources：资源与官方链接
```

## 迁移原则

1. 旧文档内容统一吸收进新结构；公开入口不再保留过时目录。
2. 新读者入口使用 Learn、Use、Build、Protocol、AI Agent、Ecosystem 六条主线。
3. 协议事实以当前代码和测试网验证为准。
4. 对外文档排除内部 debug、临时修复、Core Node 日志和实现类名。
5. GAS、AI Agent、智能合约等内容可以包含前瞻，同时区分已实现、测试中、规划中。
6. 暂不同步 `docs-en`；中文 docs 先打磨为主版本。

## 第一阶段

第一阶段目标是重建入口和叙事，而不是完成所有细节。

必须完成：

1. 新首页。
2. 新 `SUMMARY.md`。
3. Learn 入口。
4. Use 入口。
5. Build 入口。
6. Ecosystem 入口。
7. Roadmap 更新。
8. STP 作为 Protocol 中的协议白皮书，SAT20 Agent Wallet 独立进入 AI Agent。
9. Indexer 作为资产事实层进入 Learn、Build、Protocol 和 Roadmap 主线。

## 第二阶段

第二阶段重点是开发者和生态。

优先内容：

1. 智能合约开发指南。
2. GAS 资产与费用模型。
3. Indexer API、交易平台接入、L2 节点内置索引和 L1 事实分布式验证路线。
4. PWA Wallet adapter。
5. SAT20 Agent Wallet skill 测试网演练。
6. AMM、稳定币、支付、资产市场示例。

## 第三阶段

第三阶段重点是传播和增长。

优先内容：

1. 官网内容规划。
2. X 内容栏目。
3. Telegram 社区 FAQ。
4. Builder Program。
5. 生态项目展示。
6. 视频、图解、测试网证据包。

## 风格

参考比特币和以太坊文档的共同风格：

1. 少口号，多定义。
2. 少承诺，多证据。
3. 少抽象愿景，多可执行路径。
4. 把风险讲清楚，把边界讲清楚。
5. 对开发者友好，对用户诚实。
